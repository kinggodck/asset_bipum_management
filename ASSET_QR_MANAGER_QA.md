# 비품 QR 관리 프로그램 Q&A

## 1. 이 프로그램은 어떤 자료를 기준으로 만들었나요?

구글 스프레드시트의 자산관리대장과 내부 문서인 자산·비품 Naming Rule 및 QR 스티커 관리 절차서를 기준으로 만들었습니다.

초기에는 자산·장비 운영 중심으로 구성되었고, 이후 비품 등록, QR 스티커, 번호 체계, 사진 등록, 사용자 추적 기능을 추가했습니다.

## 2. 자산번호 또는 비품번호 규칙은 어떻게 되나요?

현재 적용된 번호 체계는 다음과 같습니다.

```text
[자산구분]-[품목]-[도입연월]-[순번]
```

예시:

```text
ADM-DESK-2605-001
```

의미:

- `ADM`: 일반비품
- `DESK`: 책상
- `2605`: 2026년 5월 도입
- `001`: 순번

사업장, 상세위치, 사용부서처럼 자주 바뀌는 정보는 번호에 넣지 않습니다. 이런 정보는 시스템의 상태 값으로만 관리합니다.

## 3. 위치나 부서가 바뀌면 비품번호도 바뀌나요?

아니요. 비품번호는 최초 부여 후 폐기 시까지 바꾸지 않는 고유 식별자입니다.

위치, 사용자, 사용부서가 바뀌면 스티커를 다시 발행하지 않고 시스템에서 해당 정보만 수정합니다.

## 4. 이 프로그램은 핸드폰에서도 사용할 수 있나요?

네. Vercel 등에 배포하면 핸드폰 브라우저에서도 사용할 수 있습니다.

다만 현재 `http://127.0.0.1:4177/` 같은 주소는 개발자 PC 안에서만 열리는 로컬 주소입니다. 핸드폰에서 사용하려면 Vercel 배포 URL 같은 외부 접속 가능한 주소가 필요합니다.

## 5. 앱에서 입력한 데이터는 어디에 저장되나요?

현재는 브라우저의 Local Storage에 저장됩니다.

저장 키:

```text
yg-asset-manager-v2
```

저장되는 항목:

- 비품 목록
- 사진 데이터
- 현재 사용자
- 사용 이력
- 상태 변경
- 실사일

주의할 점:

- 같은 브라우저에서는 새로고침해도 유지됩니다.
- 다른 PC나 다른 핸드폰에서는 보이지 않습니다.
- 브라우저 사이트 데이터를 삭제하면 사라질 수 있습니다.
- Google Sheet나 서버 DB에는 자동 저장되지 않습니다.

## 6. Google Sheet에서 입력한 내용은 앱에서 볼 수 있나요?

조건부로 가능합니다.

Google Sheet가 CSV로 접근 가능하도록 권한이 열려 있으면 앱에서 `시트 동기화`를 눌러 가져올 수 있습니다.

현재 흐름:

```text
Google Sheet -> 앱
```

앱에서 수정한 내용이 Google Sheet로 자동 저장되지는 않습니다.

## 7. 앱에서 입력한 내용이 Google Sheet에도 자동으로 반영되나요?

현재는 반영되지 않습니다.

앱에서 등록한 비품, 사진, 사용자 추적, 반납 이력은 브라우저 Local Storage에만 저장됩니다.

Google Sheet에 자동 반영하려면 별도 기능이 필요합니다.

필요한 기능:

- 앱에서 Google Sheet로 쓰기
- 사용 이력 전용 시트 탭 생성
- 실사 이력 전용 시트 탭 생성
- 사진은 Google Drive 등에 업로드 후 링크를 시트에 저장

권장 시트 구성:

- `비품마스터`
- `사용이력`
- `실사이력`

## 8. 앱에서 사진을 등록하면 Google Sheet에서도 볼 수 있나요?

현재는 볼 수 없습니다.

앱에서 등록한 사진은 현재 브라우저 Local Storage에 저장됩니다. Google Sheet에는 자동으로 올라가지 않습니다.

Google Sheet에서 사진을 보려면 다음 구조가 필요합니다.

1. 앱에서 사진 업로드
2. 사진을 Google Drive 또는 Vercel Blob 같은 저장소에 저장
3. 저장된 사진 URL을 Google Sheet의 `사진링크` 컬럼에 기록
4. Google Sheet에서는 링크 또는 `=IMAGE("사진URL")` 형태로 확인

사진 자체를 Google Sheet 셀에 직접 저장하기보다는 사진 URL을 저장하는 방식이 적합합니다.

## 9. 사진을 잘못 올렸을 때 수정할 수 있나요?

네. 현재 앱에서는 비품 상세 화면에서 사진을 수정할 수 있습니다.

방법:

1. 비품 목록에서 해당 비품을 선택합니다.
2. 상세 화면에서 `사진 변경`을 누릅니다.
3. 새 사진을 선택하면 기존 사진이 교체됩니다.
4. 사진을 제거하려면 `사진 삭제`를 누릅니다.

## 10. 비품을 사용한 사람을 추적할 수 있나요?

네. 현재 앱에 사용자 추적 기능이 추가되어 있습니다.

확인 가능한 항목:

- 현재 사용자
- 사용자 소속
- 사용 시작일
- 반납 예정일
- 사용 목적
- 사용 시작 이력
- 반납 이력

사용 방법:

1. 비품을 선택합니다.
2. 상세 화면의 `사용자 추적` 영역에서 사용자, 소속, 목적, 반납 예정일을 입력합니다.
3. `사용 시작`을 누르면 현재 사용자와 이력이 저장됩니다.
4. `반납 처리`를 누르면 반납 이력이 남고 현재 사용자가 초기화됩니다.

## 11. QR 스티커를 휴대폰 카메라로 찍었는데 연결이 안 됩니다. 왜 그런가요?

초기 버전의 QR 값은 예시용 단축 URL이었습니다.

예전 예시:

```text
https://cmp.ny/A001
```

이 주소는 실제 연결된 회사 도메인이나 Vercel 주소가 아니기 때문에 휴대폰으로 찍어도 열리지 않습니다.

현재 버전은 Vercel 배포 URL을 기준으로 QR 값을 생성합니다.

현재 방식:

```text
https://asset-bipum-management.vercel.app/?asset=ADM-DESK-2605-001#assets
```

따라서 새로 출력한 QR 스티커는 휴대폰 카메라로 스캔하면 앱이 열리고 해당 비품이 자동 선택됩니다.

단, 이미 출력된 예전 `cmp.ny` QR 스티커는 다시 인쇄해야 합니다.

회사 도메인이 있다면 이후 다음처럼 바꿀 수 있습니다.

```text
https://asset.company.com/assets/ADM-DESK-2605-001
```

## 12. QR 스캔 시 바로 해당 비품 상세 화면을 열 수 있나요?

네. 현재 앱은 다음 URL 파라미터를 읽습니다.

```text
?asset=ADM-DESK-2605-001
```

QR 스캔으로 앱이 열리면 해당 비품이 목록에서 자동 선택되고 상세 화면과 QR 스티커 화면에서 바로 확인할 수 있습니다.

## 13. Vercel 배포는 가능한가요?

네. Vercel 배포가 가능하도록 설정해두었습니다.

배포 준비 파일:

- `package.json`
- `vercel.json`
- `.vercelignore`
- `scripts/build.js`

배포 전 빌드:

```bash
npm run build
```

Vercel Preview 배포:

```bash
vercel
```

운영 배포:

```bash
vercel --prod
```

배포 시 내부 문서 파일은 제외하고 `dist/` 안의 앱 파일만 공개되도록 구성했습니다.

## 14. 여러 사람이 동시에 사용하려면 지금 구조로 충분한가요?

아니요. 현재 구조는 개인 브라우저 저장 방식입니다.

여러 사람이 같은 데이터를 공유하려면 중앙 저장소가 필요합니다.

가능한 저장소:

- Google Sheets 쓰기 연동
- Google Drive 사진 업로드 연동
- Supabase 또는 PostgreSQL
- Vercel Postgres
- Vercel Blob
- 사내 서버 API

실무용 권장 구조:

```text
휴대폰 QR 스캔
-> Vercel 앱 열림
-> 로그인 또는 권한 확인
-> DB 또는 Google Sheet에서 비품 조회
-> 상태, 사용자, 사진, 실사 이력 저장
```

## 15. 운영 전 반드시 결정해야 할 사항은 무엇인가요?

운영 전에 다음 항목을 결정해야 합니다.

- 최종 데이터 원장: Google Sheet 또는 DB
- 사진 저장 위치: Google Drive, Vercel Blob, 사내 파일 서버 등
- QR에 넣을 실제 URL
- 사용자 로그인 방식: 사내 SSO, Google 계정, 단순 인증 등
- 사용 이력과 실사 이력의 보관 방식
- 직원별 권한 범위
- 외부인이 QR을 스캔했을 때 보여줄 안내 페이지

## 16. 현재 앱의 성격은 무엇인가요?

현재 앱은 비품 QR 관리의 프로토타입이자 MVP입니다.

가능한 것:

- 비품 등록
- 사진 등록과 수정
- QR 스티커 출력
- 사용자 추적
- 반납 처리
- 실사일 기록
- CSV 가져오기와 내보내기
- Vercel 배포
- QR 스캔 후 상세 자동 이동

아직 운영용으로 추가해야 하는 것:

- 중앙 저장소 연동
- Google Sheet 쓰기
- 사진 파일 업로드 저장소
- 로그인과 권한 관리
- 다중 사용자 동시 사용
